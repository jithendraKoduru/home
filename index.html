üìö Understanding the Retirement

‚ùå What IS Being Retired (Critical):

Direct API calls to deprecated endpoint URLs:

  • /services/data/v21.0 through /services/data/v30.0
  • /services/Soap/u/v21.0 through /services/Soap/u/v30.0
  • /services/async/v21.0 through /services/async/v30.0

Impact: Any hardcoded URLs calling these endpoints will FAIL after Summer '25 (Release 256)

‚úÖ What is NOT Being Retired (Safe):

  • Metadata API versions in .cls-meta.xml, .trigger-meta.xml, .page-meta.xml files
  • Custom Apex Classes, Triggers, Visualforce Pages - The API version attribute in these metadata files
  • Managed Package metadata components pointing to version ‚â§30.0
  • Process Builders and Flows - API version declarations

Why? The retirement only affects runtime API endpoint calls, not metadata declarations.
